## Overview
Builtin policies are shipped under `devcovenant/builtin/policies/`.
Each policy directory defines one policy contract, optional autofix/assets,
and a sibling `test_blueprints.yaml` descriptor when the shipped tests need
to travel as package data instead of a live `tests/**` tree.
That blueprint keeps builtin tests non-discoverable by default while still
letting DevCovenant recreate the mirrored tests when a repository shadows the
builtin policy with `devcovenant custom --policy <policy-id> --do`.
`version-sync` is the reference pattern for keeping both `Project Version`
and `DevCovenant Version` headers aligned with the same canonical version
file.

Runtime behavior is sourced from AGENTS policy definitions generated by
refresh from policy descriptors plus metadata resolution layers.

## Layout Contract
Each policy directory uses this layout:
- `<policy-id>.yaml` descriptor
- `<policy-id>.py` runtime check script
- optional `test_blueprints.yaml` shipped-test descriptor
- optional `autofix/*.py`
- optional `assets/` templates

The directory name and script/descriptor basename must match policy ID.
When `test_blueprints.yaml` is present, its repository-relative paths and
serialized file contents describe the builtin test tree that should be
compared against
`tests/devcovenant/builtin/policies/<policy-id>/` and materialized into
`tests/devcovenant/custom/policies/<policy-id>/` when the policy is copied
into `devcovenant/custom/policies/<policy-id>/`.

## Descriptor Contract
Descriptors define:
- `id`
- `text` policy prose
- `metadata` defaults
- optional `test_blueprints.yaml` entries that preserve shipped tests as
  encoded descriptor data instead of a discoverable live tree

Resolved metadata is computed by runtime precedence and then written into:
- `devcovenant/registry/registry.yaml`
- AGENTS managed policy block

Example metadata-owned runtime behavior:
- `workflow_runs` are resolved from the active profile stack, recorded in
  the workflow contract, and executed through the core-owned `run`
  command surface.

## Policy Runtime Contract
Policy scripts expose policy checks and return structured violations consumed
by engine formatting and gate behavior.

Policies should remain language-agnostic. Language-aware analysis is delegated
to shared translator runtime, which returns normalized language units.

## Autofix Contract
Autofix helpers are optional.
When present, they live in `autofix/` and follow language-aware default
resolution rules:
- language-specific autofix helper when available
- `autofix/global.py` as the global default otherwise

A custom policy override suppresses builtin autofix helpers for that policy ID.

## Workflow
1. Update descriptor, runtime script, and `test_blueprints.yaml` together.
2. Keep the checked-in builtin test tree under
   `tests/devcovenant/builtin/policies/...` aligned with the blueprint.
3. Run `devcovenant custom --policy <policy-id> --do` when you need the
   repo-owned shadow copy and mirrored tests under
   `tests/devcovenant/custom/policies/...`.
4. Use `--undo` to remove that copied policy tree and mirrored tests again.
5. Keep tests behavioral/contract-focused; do not add placeholder stubs.
6. Run `devcovenant refresh` after descriptor or blueprint changes.
7. Run full gate sequence before finalizing:
   `gate --open` -> `gate --verify` (rerun until clean) ->
   `run` -> `gate --close`.
